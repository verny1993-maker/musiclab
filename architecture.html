MusicLab Pipeline Architecture

DJ set analysis: parse → download → analyze → enrich → build intelligence cards

Pipeline Stages

  • * 01_parse -- set79 tracklist to JSON
  • * 02_enrich_audio -- download + analyze
  • * enrich_library -- Discogs/Last.fm/MB metadata
  • * 03_enrich -- batch library with 8D vectors
  • * 05_build_cards -- artist & venue intelligence

Data Scale

  • * 2,388 Spotify liked tracks
  • * ~1,700 tracks analyzed (audio features)
  • * 5 DJ sets parsed
  • * 4 venues tracked
  • * 5 metadata APIs with rate limiting

Infrastructure

  • * Docker: music-analysis (librosa + essentia)
  • * SQLite WAL: audio_analysis.db
  • * Qdrant: 8D vector search
  • * yt-dlp: SC to YT fallback download
  • * Rate limiting: token bucket per API
